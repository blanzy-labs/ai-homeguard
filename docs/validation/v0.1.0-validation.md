No alternate browser-control mechanism was used.

Owner manual smoke result before Slice 12A: failed with release-blocking issues.

Findings from owner manual review:

- Site performance was fast and fluid.
- Demo Mode stayed on `Loading sample report` and did not display the sample report.
- Backend API status was visible but needed to stay useful and non-dominant.
- Navigation was not clear enough; users could feel trapped in a flow.
- User-facing safety copy used development terminology.
- Safety acknowledgement repeated between paths and would frustrate users.
- Docker on a Mac reported Linux/container visibility in a way that could be mistaken for host OS detection failure.
- Active network discovery is desired later, but must not be added to v0.1.0.

- Start Over and Clear Current Report controls that preserve the versioned safety acknowledgement
- User-facing terminology cleanup from development-slice wording to version/release wording
- Versioned frontend-only `sessionStorage` safety acknowledgement key `ai-homeguard-safety-ack-v0.1.0`
- Continued request-level Network Awareness authorization acknowledgement
- Runtime UI separation for browser/device hint, backend runtime, and audit target
- Backend container-runtime finding and limitation note for Docker/container local audits
- Documentation for Docker-on-Mac runtime visibility and native uv macOS checks
- Deferred follow-up: Slice 13 - Safe Private Network Discovery
