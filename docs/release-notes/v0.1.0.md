# AI HomeGuard v0.1.0 - Local Home Security Audit MVP

## Summary

AI HomeGuard v0.1.0 is the first local-first MVP for home cyber hygiene review. It combines guided questionnaire findings, read-only local device checks, authorization-gated Safe Private Network Discovery, passive local network awareness, manual/demo device inventory guidance, and D3FEND-informed defensive recommendations.

AI HomeGuard is part of the Blanzy Labs AI app family.

## Included Features

- FastAPI backend and React/Vite frontend
- Docker Compose support
- Dashboard-first Run HomeGuard Check flow
- Demo Mode
- Safety-first UX and authorization acknowledgements
- Home security questionnaire
- Finding/report model
- Combined HomeGuard report
- Markdown/JSON exports
- Read-only Windows local checks foundation
- Read-only macOS local checks foundation
- Read-only Linux local checks foundation
- Unified local device audit
- D3FEND-informed guidance catalog
- Passive local network awareness foundation
- Safe Private Network Discovery with private IPv4 guardrails and bounded ping-only discovery
- Device inventory helper
- Router guidance
- Report dashboard/review UI with overall status, top actions, coverage, grouped findings, evidence source labels, limitations, and export controls

## Safety Boundaries

v0.1.0 does not include:

- Nmap
- Public target discovery
- Port scanning of other devices
- Packet capture
- Router login
- Credential collection
- Public target scanning
- Automatic remediation
- Telemetry
- Database storage
- Browser persistence of questionnaire answers, reports, inventory entries, exports, or secrets
- External uploads
- AI provider calls

The frontend stores only low-risk UI state in `sessionStorage` for the current browser session: a versioned safety acknowledgement and whether Advanced Options is open.

## Known Limitations

- Not a penetration test
- Not a compliance or certification tool
- Checks are best-effort and platform-dependent
- Docker may reflect the container environment rather than the host; Docker on a Mac can report Linux container runtime visibility
- Windows and Linux checks should be further validated on native systems
- Passive network awareness may be incomplete
- Safe Private Network Discovery may be incomplete when Docker, VPNs, guest Wi-Fi, client isolation, or local firewalls limit visibility
- Manual device inventory depends on user-provided or router app information
- D3FEND-informed guidance is educational and does not represent official D3FEND certification or full D3FEND coverage

## Post-v0.1.0 Direction

Slice 13 moved AI HomeGuard toward a dashboard-first experience for non-technical home users. The recommended path is now one `Run HomeGuard Check` flow, with Demo Mode, Local Device Audit, Questionnaire, Local Network Awareness, and Device Inventory Helper presented as parts of one product rather than unrelated tools.

Slice 14 added safe private network device discovery for the dashboard-first flow. Future dashboard work should continue improving simple next steps and reducing technical choices before value.
